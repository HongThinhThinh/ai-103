Microsoft Foundry

Microsoft Foundry is the recommended Azure platform for organizing AI projects, resources, models, agents, tools, knowledge, endpoints, and SDK-driven automation.

Platform

Foundry projects

Manage resource connections, data, code, models, agents, tools, and knowledge under a Foundry resource. A resource can support multiple projects.

Foundry portal and SDK

The portal supports visual development and testing. The SDK supports scripted project automation and CI/CD workflows.

Models

Deploy LLMs from Foundry Models and consume them through Foundry or Azure OpenAI endpoints.

Agents

Named AI configurations that combine an LLM, instructions, and tools for autonomous task automation.

Knowledge

Foundry IQ can centralize MCP-based knowledge connections so agents can ground responses.

Select, Deploy, and Evaluate Models

Foundry Models provides a catalog with more than 1,900 models. You can filter, compare benchmarks, deploy endpoints, test in playgrounds, and evaluate quality and safety.

Model lifecycle

Explore the catalog

Search by provider, collection, capability, inference task, fine-tuning method, industry, and deployment options.

Compare benchmarks

Use quality, safety, cost, and performance metrics to choose a model that fits your scenario and budget.

Deploy to endpoints

Choose deployment types such as Global Standard, Provisioned, Batch, Data Zone, Regional, or Developer deployments.

Test in playground

Try prompts, system messages, temperature, max tokens, and code samples before application integration.

Evaluate performance

Use manual reviews, structured test cases, automated quality metrics, risk metrics, NLP metrics, and evaluator libraries.

Key benchmark dimensions: quality, safety, cost per token, latency, throughput, time to first token, and scenario-specific leaderboards.

Generative AI Apps That Use Tools

Tools extend a model beyond static training data so it can retrieve current information, search private files, run code, or call application functions.

Responses API

code_interpreter

Runs Python in a sandbox for calculations, data analysis, file processing, and algorithm prototyping.

web_search

Retrieves current web information for source-aware answers about changing facts, current events, or policies.

file_search

Searches uploaded files in vector stores to answer from trusted private documents with grounded context.

function

Lets the model request developer-defined functions while your application controls execution, validation, and outputs.

Optimize Generative AI Performance

Start with prompt engineering. Add RAG when the model needs trusted knowledge. Add fine-tuning when the model needs consistent behavior, style, or format.

Prompt + RAG + Fine-tune
StrategyUse WhenStrengthTrade-off
Prompt engineeringYou need tone, role, format, guardrails, examples, or parameter tuning.Fast, low complexity, low cost.Long prompts cost tokens and may not guarantee compliance.
RAGThe model needs private, current, factual, or domain-specific data.Improves accuracy by retrieving trusted context.Requires search indexes, embeddings, storage, and data maintenance.
Fine-tuningYou need reliable style, format, tool use, distillation, or reduced prompt length.Embeds desired behavior into model weights.Higher cost, training data needs, maintenance, and experimentation.
Decision flow: start simple with prompts, add RAG for factual grounding, add fine-tuning for consistent behavior, and combine strategies only when the application requires it.

Responsible Generative AI

Responsible AI requires identifying risks, testing outputs, reducing harms across layers, and managing release and operations with clear governance.

Map, Measure, Mitigate, Manage

Map

Identify, prioritize, test, verify, document, and share potential harms such as offensive content, inaccuracies, or unsafe guidance.

Measure

Create diverse prompts, generate outputs, categorize harm levels, and establish a baseline for improvement.

Mitigate

Apply layered mitigations at the model, safety system, system message and grounding, and user experience layers.

Manage

Complete legal, privacy, security, and accessibility reviews. Plan phased release, incidents, rollback, feedback, and telemetry.

Microsoft Responsible AI principles

  • Fairness
  • Reliability and safety
  • Privacy and security
  • Inclusiveness
  • Transparency
  • Accountability

Operational safeguards

  • Block harmful responses quickly
  • Block abusive users or IP addresses
  • Collect user feedback and issue reports
  • Track privacy-compliant telemetry
  • Maintain incident and rollback plans
